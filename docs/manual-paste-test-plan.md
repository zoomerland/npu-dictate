# Manual Paste Test Plan

Цель: проверить не качество длинной диктовки, а именно надежность вставки в разные типы полей. Для этих тестов лучше говорить короткие фразы, чтобы ASR не смешивался с проблемами фокуса и вставки.

## Подготовка

- Запусти приложение и дождись состояния "Готово".
- В настройках оставь включенными:
  - "Вставлять в активное поле".
  - "Восстанавливать текстовый буфер после вставки".
  - "Учитывать текст перед курсором".
  - "Добавлять пробел после вставки по контексту".
- Для браузерных тестов открой локальную страницу:
  - `tools/paste_test_page.html`
- Если вставка не произошла, проверь:
  - появился ли текст в буфере обмена;
  - появилась ли строка `paste failed` или `target_ready=False` в диагностике;
  - не ушел ли фокус в другое окно.

## Тестовые фразы

Используй короткие фразы, по одной записи на пункт:

- `Проверка вставки номер один.`
- `Это продолжение текущей строки.`
- `Казнить нельзя помиловать.`
- `Фраза после паузы должна вставиться сюда.`
- `Быстрый короткий тест.`

## Как записывать результат

Для каждого пункта достаточно:

- `OK` - текст сам вставился в нужное поле.
- `CLIPBOARD` - текст только скопировался в буфер, но не вставился.
- `WRONG_TARGET` - текст вставился не туда.
- `OVERWRITE` - текст неожиданно заменил существующий текст.
- `FOCUS_BACK` - фокус визуально вернулся, но вставки не было.

Дополнительно отмечай приложение, режим записи (`hold` или `toggle`) и короткое описание.

## Reasonable Downloadable Apps

Эти приложения не обязательны все сразу. Достаточно пройти 2-3 из них, если они уже установлены или их удобно поставить.

| ID | Приложение | Почему тестируем | Result |
| --- | --- | --- | --- |
| A1 | VS Code или VSCodium | Electron/editor input, частый рабочий сценарий | |
| A2 | Telegram Desktop | Популярное desktop-поле ввода сообщений | |
| A3 | Discord | Electron chat input | |
| A4 | LibreOffice Writer | Бесплатный Word-like редактор | |
| A5 | Obsidian | Electron markdown editor | |

Для каждого приложения:

1. Пустое поле: надиктовать короткую фразу.
2. Конец существующей строки: надиктовать продолжение.
3. Середина строки: вставить короткий фрагмент.
4. Переключиться в другое окно и обратно, повторить вставку.

## Что чинить первым

Приоритеты багов:

1. `OVERWRITE` - самое опасное, может уничтожить пользовательский текст.
2. `WRONG_TARGET` - вставка в старое окно или вкладку.
3. `CLIPBOARD` в обычных полях Chrome/Notepad.
4. `FOCUS_BACK` без вставки.
5. Лишние пробелы и мелкая пунктуация.

## После завершения

Когда матрица вставки станет стабильной, переходим к отдельному плану длинной диктовки: длинные фразы, паузы, быстрый темп, случайная тишина и отмена записи.
